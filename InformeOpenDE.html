Clemente Mujica.

  1. Descripción.

Hacer un programa que actúe como agenda virtual, la cual permita organizar los horarios del laboratorio de computadores del edificio P. Este permitirá organizar los horarios por sala.
Permite adaptar un robot a nuevos programas educativos y adaptar un programa a nuevas plataformas robóticas.

  1. Análisis del Problema.

Actualmente en la universidad no hay un sistema para organizar fácilmente los horarios de las salas de laboratorios en el edifico del P.

Este programa se encarga de organizar los horarios de manera visual, permitiendo colocar o sacar la asignatura en el día, bloque y sala deseada.

-

Caso de Uso N°1

Nombre: No ingresa asignatura.

Propósito: Agregar asignatura de manera rápida, sin importar la asignatura.

Actor: Administrador.

Pre-condición: El programa esta correctamente inicializado.

Evento: Aprieta el botón agregar, se abre una ventana de dialogo que muestra las opciones, aprieta aceptar y se agrega la primera asignatura de la lista de asignaturas del programa.

Post-condición: La asignatura se queda guardada en el programa.

Tipo: manual.

Caso de Uso N°2

Nombre: Agregar un horario que ya esta ocupado.

Propósito: Agregar asignatura en un horario sin saber que esta ocupado.

Actor: Administrador.

Pre-condición: El bloque horario que se quiere escoger ya está ocupado con un horario.

Evento: El usuario ingresa el horario con el mismo cuadro de dialogo y se agrega el horario deseado, el programa muestra una ventana de error diciendo que el horario ya está ocupado.

Post-condición:El programa seguirá teniendo el mismo horario anterior, dejando al usuario elegir de nuevo el horario que desea.

Tipo: Automatico.

Caso de Uso N°3

Nombre: Relación entre bloque y horario.

Propósito: Saber que horario corresponde el bloque de clases.

Actor: Administrador.

Pre-condición: El programa inicializado correctamente..

Evento: El usuario mueve el mouse sobre el bloque y este cambia para que se vea su correspondiente horario.

Post-condición: El programa sigue igual, solo cambia que cuando se saque el mouse del bloque se mostrara de nuevo el bloque.

Tipo: Automático.
